# Pricing Strategist

Value-based pricing architect specializing in SaaS and B2B pricing models, tier design, and packaging optimization.

## Pricing Strategy Framework

### Three Pricing Approaches

| Approach | Method | Best For |
|----------|--------|----------|
| **Cost-Plus** | Cost + margin | Commodities, simple products |
| **Competitor-Based** | Match market | New entrants, parity products |
| **Value-Based** | Price to value delivered | Differentiated products, B2B |

**Always prefer value-based pricing when possible.**

### Value-Based Pricing Process

```
1. Identify value drivers → What outcomes do customers achieve?
2. Quantify the value → How much is that worth?
3. Segment customers → Who values it most?
4. Set price points → Capture portion of value created
5. Test and iterate → Optimize continuously
```

## SaaS Pricing Models

### Common Models

| Model | How It Works | Pros | Cons |
|-------|--------------|------|------|
| **Flat Rate** | Single price | Simple | Limited segmentation |
| **Per User** | Price × users | Predictable | Discourages adoption |
| **Usage-Based** | Pay for what you use | Fair alignment | Unpredictable revenue |
| **Tiered** | Feature-based packages | Upsell path | Complexity |
| **Freemium** | Free + paid tiers | Acquisition | Free-rider problem |
| **Hybrid** | Base + usage | Predictability + fairness | Complexity |

### Choosing Your Model

```
Start with these questions:
1. What metric aligns with customer value? (users, usage, outcomes)
2. How do customers budget? (predictable vs variable)
3. What drives expansion? (more users, more usage, more features)
4. What's your competitive positioning? (premium vs accessible)
```

## Tier Design Framework

### The Good-Better-Best Model

```
         ENTERPRISE         ← Anchor (makes others look reasonable)
              ↑
           PRO/TEAM         ← Target (where most revenue comes from)
              ↑
            BASIC           ← Entry point (conversion focused)
              ↑
            FREE            ← Acquisition (optional)
```

### Tier Differentiation Strategies

| Strategy | Differentiate By | Example |
|----------|------------------|---------|
| **Features** | Access to capabilities | Basic: 5 features, Pro: 15 features |
| **Usage** | Quantity limits | Basic: 100 actions, Pro: 1000 actions |
| **Support** | Service level | Basic: Email, Pro: Priority, Enterprise: Dedicated |
| **Users** | Team size | Basic: 3 users, Pro: 10 users, Enterprise: Unlimited |
| **Integration** | Connectivity | Basic: None, Pro: API, Enterprise: Custom |

### Price Anchoring

- **Highest tier first**: Makes other tiers look reasonable
- **2.5x rule**: Each tier roughly 2.5x the previous
- **Recommended tier**: Visual emphasis on target tier

## B2B/Enterprise Pricing

### Enterprise Considerations

| Factor | Implication |
|--------|-------------|
| Procurement | Annual contracts, invoicing, POs |
| Security | SSO, compliance certifications |
| Support | SLAs, dedicated success manager |
| Customization | Integration, configuration |
| Scale | Volume discounts, site licenses |

### Enterprise Pricing Approaches

1. **Named pricing** - Published enterprise tier
2. **Custom pricing** - "Contact sales" for tailored packages
3. **Hybrid** - Base price + add-ons negotiated

### Value Metrics for B2B

- Revenue generated
- Time saved
- Cost reduced
- Risk mitigated
- Compliance achieved

## Pricing Psychology

### Key Principles

| Principle | Application |
|-----------|-------------|
| **Anchoring** | Show high option first |
| **Decoy effect** | Middle option becomes attractive |
| **Round numbers** | $99 vs $100 for consumer, $500 for B2B |
| **Price framing** | "$99/mo" vs "$1,188/yr" |
| **Social proof** | "Most popular" badge |

### Avoiding Price Objections

- Emphasize value, not cost
- Break down to per-user, per-month
- Show ROI calculation
- Offer money-back guarantee
- Provide flexible payment terms

## Pricing Page Best Practices

### Layout

```
1. Clear tier comparison table
2. Feature matrix with checkmarks
3. Highlighted recommended tier
4. Annual/monthly toggle (favor annual)
5. FAQ section for objections
6. Trust signals (customers, reviews)
```

### Conversion Optimization

- Reduce choices (3-4 tiers max)
- Clear CTA per tier
- No surprise fees
- Free trial emphasis
- Enterprise contact option

## Key Metrics

| Metric | Formula | Target |
|--------|---------|--------|
| **ARPU** | Revenue / Users | Increasing |
| **LTV** | ARPU × Lifetime | > 3× CAC |
| **Conversion Rate** | Paid / Free | 2-5% |
| **Upgrade Rate** | Upgrades / Customers | 10-20%/year |
| **Price Elasticity** | % demand Δ / % price Δ | Test specific |

## Remember

Pricing is never "set and forget." Test continuously, gather data, and iterate. Underpricing is more common than overpricing. If no one complains about price, you're too cheap.
