# App Store Optimization (ASO) Guide

Optimize app visibility and downloads in App Store and Google Play.

## When to Use

- Launching a new mobile app
- Improving app store rankings
- Optimizing app metadata
- Creating effective screenshots
- Responding to reviews

## Platform Differences

| Element | App Store (iOS) | Google Play |
|---------|-----------------|-------------|
| Title | 30 characters | 30 characters |
| Subtitle | 30 characters | N/A |
| Short Description | N/A | 80 characters |
| Description | 4000 characters | 4000 characters |
| Keywords | 100 characters | In description |
| Screenshots | Up to 10 | Up to 8 |

## Keyword Research

### Process

1. **Brainstorm** seed keywords
2. **Expand** with competitor analysis
3. **Evaluate** search volume and difficulty
4. **Prioritize** by opportunity score
5. **Test** and iterate

### Keyword Types

| Type | Example | Use |
|------|---------|-----|
| Brand | "spotify" | Low competition |
| Generic | "music player" | High volume |
| Long-tail | "offline music player" | Low difficulty |
| Competitor | "pandora alternative" | Strategic |

## Metadata Optimization

### Title Formula

```
[Brand Name] - [Primary Keyword] [Value Prop]
```

Example: "Calm - Meditation & Sleep"

### Subtitle (iOS)

```
[Secondary Keyword] | [Benefit]
```

Example: "Relax & Focus | Daily Mindfulness"

### Description Structure

```
[Hook - first 1-2 sentences visible]

Key Features:
• [Feature 1 with keyword]
• [Feature 2 with keyword]
• [Feature 3 with keyword]

Why Users Love Us:
• [Social proof]
• [Award/recognition]

[Call to action]
```

## Screenshot Best Practices

### Layout Options

1. **Feature-focused**: Show key feature with caption
2. **Benefit-driven**: Show outcome/result
3. **Social proof**: Include ratings/testimonials
4. **Comparison**: Before/after or vs competitors

### Caption Guidelines

- Keep under 5 words
- Use action verbs
- Highlight benefits
- Include keywords naturally

### Order Strategy

1. Most compelling feature
2. Unique differentiator
3. Popular feature
4. Social proof
5. Call to action

## Review Management

### Response Template: Positive

```
Thank you for the wonderful review, [Name]!
We're thrilled that [specific feature] has been helpful.
Your support means a lot to our team!
```

### Response Template: Negative

```
Hi [Name], thank you for your feedback.
We're sorry to hear about [specific issue].
Our team is looking into this. Please reach out to
[support email] so we can help resolve this for you.
```

### Review Strategy

- Respond within 24-48 hours
- Personalize responses
- Address specific concerns
- Avoid defensive tone
- Turn negatives into positives

## A/B Testing

### What to Test

| Element | Impact | Effort |
|---------|--------|--------|
| App Icon | High | High |
| Screenshots | High | Medium |
| Title | Medium | Low |
| Description | Low | Low |

### Testing Process

1. Create hypothesis
2. Design variants
3. Split traffic (Google Play Console)
4. Run for 7+ days
5. Analyze with statistical significance
6. Implement winner

## Metrics to Track

| Metric | Formula | Benchmark |
|--------|---------|-----------|
| Impression to View | Views / Impressions | 5-10% |
| View to Download | Downloads / Views | 25-35% |
| Conversion Rate | Downloads / Impressions | 2-5% |
| Keyword Rankings | Position for target keywords | Top 10 |
